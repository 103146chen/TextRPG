├── story.json          ← 故事設定（角色、結局、起始場景）
├── scenes/             ← 所有場景的 Markdown 檔

├── bgm/                ← 背景音樂
└── se/                 ← 音效在瀏覽器打開故事：

}三、場景檔格式每個場景是一個 .md 檔案，分成 front matter（設定）和本文（劇情）兩部分。Front matter（場景設定）markdown---
id: 場景的唯一ID
title: 場景標題（目前不顯示）
background: 預設背景圖.svg
bgm: 背景音樂.mp3
next: 下一個場景的ID
---
id 要和檔名一致，例如 01_開場.md 的 id 就是 01_開場
background 指定的圖會在場景開始時自動切換，不需要在本文第一行再寫一次 @bg
next 是這個場景跑完後自動跳到的下一場。如果場景結尾有 @goto 或 @ending，可以不填
四、本文指令文字顯示markdown角色名: 「對話內容。」

這樣直接寫就是旁白，不屬於任何角色的聲音。角色名會顯示在文字框上方，旁白不顯示角色名。畫面指令（以 @ 開頭）背景markdown@bg 圖檔名.svg fade=800fade= 是淡入毫秒數（省略不寫也可以，預設 800ms）。注意：scene 開始時 front matter 的 background: 已經會切換背景，不要在本文開頭重複寫同一張。 只有在場景中途要切換到不同背景時，才在本文寫 @bg。角色立繪markdown@show 阿秀 center normal
@show 阿秀 left worried
@hide 阿秀
@hide all位置只有三種：left、center、right。表情要對應 story.json 裡定義的 key。音樂音效markdown@bgm 音樂檔.mp3
@bgm-stop
@se 音效.mp3等待markdown@wait 1500單位是毫秒，在畫面切換之間停頓用。設定 flagmarkdown@set 選擇 = "A"
@set 好感度 += 1
@set 存款 -= 3
flag 名稱可以使用中文（例如 借錢對象、籌款結果）
值可以是：字串（"加引號"）、數字（1）、布林（true / false）
運算子：= 直接設值、+= 加、-= 減
條件分支markdown:: if 借錢對象 == "岳家"
岳父嘆了口氣。
:: elif 借錢對象 == "廟口"
廟祝翻了翻帳冊。
:: else
你找不到人借。
:: endif
if、elif、else、endif 都以 :: 開頭
條件運算子：==、!=、>、<、>=、<=
可以用 &&（且）、||（或）、!（否）組合條件
巢狀 if 可以，但建議不要超過兩層
跳轉markdown@goto 另一個場景ID
@goto 同場景內的標籤名搭配標籤使用：markdown:: label 匯集點

這段文字不管前面選什麼都會看到。玩家選項markdown?? choice
- text: 選項文字A
  set: { flag名: 值 }
  goto: 目標
- text: 選項文字B
  set: { flag名: 值 }
  goto: 目標
??
set: 和 goto: 都是選用的
goto: 可以是場景 ID 或同場景的標籤名
多個選項可以設同一個 flag，用不同的值區分玩家的選擇
常見模式：選完之後匯集到同一個標籤繼續：markdown?? choice
- text: 選A
  set: { 路線: "A" }
  goto: after_choice
- text: 選B
  set: { 路線: "B" }
  goto: after_choice
??

:: label after_choice

:: if 路線 == "A"
走了A的路。
:: else
走了B的路。
:: endif

後面的共同劇情接著走。結局markdown@ending id="ending_A" title="結局標題" desc="圖鑑說明"這個指令會結束遊戲，跳到標題畫面，並解鎖圖鑑。id 要和 story.json 裡 endings 列表的 id 對應。五、場景之間的跳轉邏輯引擎處理場景切換的順序：
場景跑到最後一行 → 自動跑 front matter 的 next: 場景
場景中出現 @goto 場景ID → 跳到指定場景
選項的 goto: → 跳到指定場景或標籤
出現 @ending → 結束遊戲
