一、選項不是「對或錯」，是「揭示角色」
玩家選的每一個選項，不是在「通關」，而是在回答一個問題：這個人是什麼樣的人？
以秦得參劇本為例，看到巡查拿走秤仔時，四個選項：

老實承認 → 這個人不懂得保護自己
裝傻 → 這個人學會了用弱者的方式周旋
辯解 → 這個人還沒被壓垮，仍然相信道理
送菜 → 這個人試圖用「給」來換取「不被拿」

沒有一個是「正確答案」。每個都照見了不同的得參——而四個選項放在一起，讓學生看見：殖民地的台灣人面對這種情境，每條路都走得通，每條路都有代價。

原則一：讓選項揭示世界觀，而不是測試玩家


二、選項的三種功能
一個選項同時可以有一到三種功能，但功能越多、設計難度越高：
功能 A：改變後續敘述（口吻、細節）
選擇影響接下來幾段文字的描述方式，但不改變劇情走向。最容易寫、效果也夠好。
?? choice
- text: 跪下來求情
  set: { 面對罰金: "下跪" }
  goto: after_choice
- text: 咬牙站著說「好，我去籌」
  set: { 面對罰金: "硬吞" }
  goto: after_choice
??

:: label after_choice

:: if 面對罰金 == "下跪"
巡查看著你下跪的樣子，嘴角勾起。
:: else
巡查盯著你看了一會，似乎有點意外。
:: endif

不管哪種，他拿著秤走了。
同樣的結果（秤被拿走），但玩家感受到的屈辱方式不同。
功能 B：埋下伏筆（稍後影響）
選擇設一個 flag，幾個場景後用 :: if 讀取，讓玩家覺得「之前的選擇有意義」。
# 第四章借錢
- text: 讓阿秀典當耳環
  set: { 借錢對象: "典當" }

# 第七章、第九章讀取這個 flag
:: if 借錢對象 == "典當"
你看著阿秀的耳垂——空的。你沒有讓她贖回來。
:: endif
伏筆的力量來自時間差。學生在第四章選「典當」的時候，不知道這個選擇會在第九章除夕夜回來找他們。
功能 C：決定結局（分流）
最後的大選擇才用這個功能。不要把分流藏在劇情中段，學生會迷失。把分流集中在最後一兩個場景，前面的選擇都只做功能 A 和 B。
# 第九章：唯一真正分流的選項
?? choice
- text: 握緊秤桿，往派出所走
  goto: 10a_同歸
- text: 把秤桿放回地上，走回家
  goto: 10b_苟活
??

三、「怎麼選都沒有好結果」的設計方法
秦得參劇本的核心教學目標就是讓學生體會這件事。做法：
選項要有「邏輯上說得通的理由」才會讓人真的選
壞的選項設計：

「向巡查求情」
「逃跑」
「打他」

這三個放在一起，玩家會本能地選「逃跑」或「打他」，覺得「求情才是輸家」。但這不是真正的困境。
好的選項設計：每個選項都要讓人覺得我可能選對了，然後才讓結果告訴他其實沒有。
巡查拿走秤的場景：

「老實說沒驗過」→ 聽起來最誠實，應該最安全
「裝傻」→ 這招對付官員好像有用
「辯解」→ 我的秤真的是準的，說清楚應該沒事
「送菜」→ 息事寧人，給他個甜頭就好了

每一個在選的瞬間都像是「這個可能行得通」，但四個都行不通。學生選完才發現：不是選錯了，是這個世界沒有給你對的選項。

原則二：讓每個選項在選的瞬間都合理，讓後果告訴他不合理

後果的輕重要有層次
不是所有選擇都造成同樣嚴重的後果。用層次製造「遞進感」：
選項後果老實回答秤被拿走，罰三塊裝傻秤被拿走，罰三塊，巡查更兇辯解秤被拿走，罰三塊，被罵「反駁警察」送菜秤被拿走，罰三塊，被說成「行賄」，再多丟一顆高麗菜
都是壞結果，但程度不同。學生會發現：越是想「聰明地應對」，往往越慘。

四、flag 命名與管理
每個 flag 建議在一個地方列清楚，因為場景一多，很容易打錯。
命名規則
用動作或事件命名，不要用抽象狀態：
不好好flag1借錢對象choice_3面對巡查的方式status籌款結果
flag 的四種用途
1. 記錄玩家的選擇（字串值）
@set 借錢對象 = "岳家"
讀取時：
:: if 借錢對象 == "岳家"
2. 記錄是否發生過某件事（布林值）
@set 看見了耳環 = true
讀取時：
:: if 看見了耳環
3. 累積數值（數字）
@set 欠債金額 += 3
讀取時：
:: if 欠債金額 > 5
你欠的已經還不完了。
:: endif
4. 伏筆 flag（用布林記錄某個深層選擇）
@set 埋伏筆_母親遺言 = true
在很後面的場景才讀取，加深敘事厚度。

五、選項的數量
每個選擇點 2–4 個選項最佳。超過 4 個，玩家的認知負擔太高，會隨便選；少於 2 個，就不是真的選擇。
情況建議人生觀念的選擇3 個（三種不同的世界觀）面對困境的即時反應4 個（涵蓋逃、抗、忍、妥協）最終結局分流可以到 6–7 個，但每個要有意義純粹推進劇情2 個就夠，甚至可以不用選項

六、什麼時候不要放選項
不要為了放選項而放選項。這幾種情況不需要：

結果不管選什麼都一樣 → 直接寫旁白就好，不要假裝給選項
選項只是「要不要繼續」 → 這不是選擇，是點擊繼續
選項在敘事節奏很快的時候出現 → 打斷感情的積累

秦得參劇本裡，阿母臨終那一段沒有選項——只有旁白和對話。因為那一刻不需要玩家「選擇」，需要他們「感受」。選項放在那裡反而會切斷情緒。

原則三：選項要放在玩家真正需要做決定的地方


七、讓選項有教學意義
針對文學教學的特殊設計建議：
把「貼合原文」與「不貼合原文」混在一起
?? choice
- text: 忍氣吞聲，把秤交出去         ← 貼合原文的選擇
- text: 裝傻說不懂規定               ← 合理延伸
- text: 辯解秤是準的                 ← 合理延伸
- text: 把菜送他，希望他放過         ← 合理延伸，但結果更慘
??
學生看到「第一個選項」，會想：「這就是書裡寫的。」
他們也可能選其他的，然後發現：「就算不是書裡的選擇，結果也好不到哪裡去。」
兩種路徑都有教學效果——一種讓他們認識原文，另一種讓他們理解「不是他選錯，是環境不給他對的選項」。
讓「看起來最聰明」的選項不是最好的
送菜給巡查，是讀過書、懂得「用禮物換關係」邏輯的人可能想到的。結果比老實說還慘——被冠上「行賄警察」的名義。
這個設計讓學生體驗一件事：用現代人的思維去解殖民時代的問題，會完全失效。 這比老師講十遍「當時台灣人沒有人權」更有力。

八、快速設計一個選項的流程
碰到一個劇情點，不知道要不要放選項，問自己三個問題：
1. 這個時刻，真實的人會猶豫嗎？
如果答案是「是」，值得放選項。如果在現實裡只有一條路，放選項就是假的。
2. 不同的選擇，揭示的是同一種人還是不同種人？
如果選A和選B揭示的都是「害怕的人」，只要一個選項就夠了（或者不要選項）。不同的選項應該對應不同的應對姿態。
3. 我有辦法寫出每個選項的後果嗎？
如果想不出「選了B之後接下來怎麼樣」，那個選項還不成熟，先不要放。

九、秦得參劇本的分支地圖（給你參考用）
01 童年
  └─ 童年印象（疑惑/畏懼/憤怒）       ← 功能A，改變敘述

02 少年
  └─ 工作選擇（糖廠/苦力/長工/私菸）   ← 功能A+B，影響後面一兩段

03 成婚
  └─ 面對喪母（承擔/空虛/憤怒/遺言）   ← 功能B，埋伏筆

04 買秤
  └─ 借錢對象（岳家/廟口/典當/頭家）   ← 功能B，多處後續有影響

05 第一天（無選項）

06 巡查
  └─ 秤仔回應（老實/裝傻/辯解/送菜）   ← 功能A，微調後續

07 衝突
  └─ 面對罰金（下跪/勞役/硬吞/想搶/母親的話）  ← 功能A+B

08 籌錢
  └─ 籌款方式（打零工/求頭家/求巡查/變賣/放棄） ← 功能B，影響除夕敘述

09 除夕（唯一真正分流）
  └─ 最終決定 → 7 個結局              ← 功能C，分流
前八個選擇影響敘事的細節和情感厚度，最後一個選擇決定結局。這是刻意的設計——讓學生覺得前面每一步都有意義，但到最後才知道真正的抉擇在哪裡。
